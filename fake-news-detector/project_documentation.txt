1. EXECUTIVE SUMMARY
================================================================================

The Fake News Detector is a machine learning application designed to classify news
articles into three categories: Fake, Real, or Satire. Utilizing natural language
processing techniques and a TF-IDF vectorizer combined with Logistic Regression,
the system provides both web-based and API-based interfaces for real-time news
analysis.

Key Features:
- Real-time news classification
- Web-based user interface (Streamlit)
- REST API for programmatic access
- Model explainability features
- Configurable preprocessing pipeline
- Support for multiple ML models

================================================================================
2. INTRODUCTION
================================================================================

2.1 PURPOSE
-----------
In an era of information overload, distinguishing between credible news and
misinformation is crucial. This project aims to provide an accessible tool for
automated news verification using machine learning techniques.

2.2 SCOPE
---------
The system covers:
- Text-based news article classification
- Three-class classification (Fake/Real/Satire)
- Web interface for individual predictions
- API for batch processing and integration
- Model training and evaluation pipelines

Out of scope:
- Image/video analysis
- Multi-language support (English only)
- Real-time news monitoring
- Social media integration

2.3 OBJECTIVES
--------------
- Achieve >80% classification accuracy
- Provide user-friendly web interface
- Enable easy API integration
- Support model explainability
- Maintain extensible architecture

5. INSTALLATION GUIDE
================================================================================

Prerequisites:
1. Python 3.8+ installed
2. Git (optional, for cloning repository)

Installation Steps:

1. Download/Clone the project:
   git clone [repository-url]
   cd fake-news-detector

2. Create virtual environment:
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/macOS:
   source .venv/bin/activate

3. Install dependencies:
   pip install -r requirements.txt

4. Download required NLTK data:
   python -c "import nltk; nltk.download('wordnet'); nltk.download('stopwords')"

5. Verify model file exists:
   - Check models/tfidf_logreg.joblib exists
   - If missing, train model (see Section 9.1)

6. Test installation:
   python -c "import streamlit, sklearn, pandas; print('Installation successful')"

6. USER MANUAL
================================================================================

6.1 WEB INTERFACE
-----------------

Starting the Application:
1. Activate virtual environment
2. Run: streamlit run ui/streamlit_app.py
3. Open browser to http://localhost:8506

Using the Interface:
1. Enter article title (optional)
2. Enter article body text
3. Click "Predict" button
4. View classification result and confidence score
5. Optionally explore model explanations

Features:
- Dark mode toggle
- Model selection (TF-IDF LR, BERT coming soon)
- Feature importance visualization (Word Cloud/Bar Chart)

6.2 API USAGE
-------------

Starting the API:
1. python src/api/app.py
2. API available at http://localhost:5000

API Endpoints:

POST /predict
Request Body:
{
  "title": "Article Title (optional)",
  "body": "Article content (required)"
}

Response:
{
  "prediction": "Fake|Real|Satire",
  "confidence": 0.95,
  "probabilities": {
    "Fake": 0.02,
    "Real": 0.95,
    "Satire": 0.03
  }
}

Example using curl:
curl -X POST http://localhost:5000/predict \
     -H "Content-Type: application/json" \
     -d '{"body": "This is a sample news article..."}'

7. TECHNICAL SPECIFICATIONS
================================================================================

7.1 ARCHITECTURE
----------------
The system follows a modular architecture:

- UI Layer: Streamlit web application
- API Layer: Flask REST API
- Processing Layer: Text preprocessing pipeline
- Model Layer: Scikit-learn ML models
- Data Layer: CSV-based data storage

7.2 DATA PROCESSING PIPELINE
----------------------------
1. Text Input → 2. Preprocessing → 3. Feature Extraction → 4. Prediction

Preprocessing Steps:
- Lowercase conversion
- Punctuation removal
- Stopword filtering
- Lemmatization
- TF-IDF vectorization (max_features=5000)

7.3 MODEL DETAILS
-----------------
Current Model: TF-IDF Vectorizer + Logistic Regression

Parameters:
- TF-IDF: max_features=5000, ngram_range=(1,2)
- Logistic Regression: C=1.0, max_iter=1000

Performance Metrics (approximate):
- Accuracy: 85%
- Precision: 83%
- Recall: 84%
- F1-Score: 83%

11. TROUBLESHOOTING
================================================================================

Common Issues:

1. Import Errors:
   - Ensure all dependencies installed
   - Check Python version compatibility
   - Verify virtual environment activation

2. Model Loading Errors:
   - Check models/tfidf_logreg.joblib exists
   - Retrain model if corrupted
   - Verify scikit-learn version compatibility

3. API Connection Issues:
   - Check port 5000 availability
   - Verify firewall settings
   - Test with different host (127.0.0.1 vs localhost)

4. Memory Errors:
   - Reduce max_features in configuration
   - Process data in batches
   - Increase system RAM

5. UI Not Loading:
   - Clear browser cache
   - Try different browser
   - Check Streamlit version
